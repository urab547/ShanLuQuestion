## 1. 游戏概述

### 1.1 基本信息

| 字段 | 内容 |
|------|------|
| **游戏名称** | 山麓之问 |
| **游戏类型** | 叙事解谜 × 环境探索 × AI氛围对话 |
| **平台** | 网页端（PC为主，移动端自适应） |
| **单章时长** | 30–45 分钟 |
| **文化场域** | 藏族 · 拉萨周边村落 · 唐卡传承 |
| **时代背景** | 2020年前后 |
| **核心动词** | 发现 / 追问 |
| **核心情绪** | 敬畏 + 好奇 + 不舍 |

### 1.2 一句话定位

一款以追查失落唐卡为线索，通过与AI驱动的藏族村民深度对话，在信仰、传统与现代的张力中拼凑出一段被遗忘故事的叙事解谜游戏。

### 1.3 核心设计原则

**解谜驱动主线，AI对话驱动氛围。两者职责清晰、互不替代：**

- **解谜系统负责推进剧情**——玩家必须通过解谜才能进入下一阶段
- **AI对话负责让世界活着**——玩家可随时和NPC自由聊天，聊文化、聊日常、聊村子的事，这些对话不影响主线进度，但能大幅加深沉浸感
- **文化知识是解谜的钥匙**——玩家必须真正理解藏族文化才能破解谜题，不是做题，是理解

### 1.4 游戏感觉

*玩家是一个跋涉至藏区小村的寻访者。天地辽阔，人很渺小。没有人主动迎接你，也没有人驱逐你。你只是站在这里，风从山口吹来，玛尼堆的石头上刻着你看不懂的字。你来这里是为了找什么？这里的人也想知道。*

### 1.5 游戏循环结构

**内层循环（每次坐下来玩，5–15分钟）**

```
发现线索/谜题 → 探索场景寻找材料或信息 → 解开谜题
→ 获得新线索或NPC态度变化 → 发现新的问题
```

每次内层循环结束时，玩家手里必须多了一样东西：材料、信息、关系或新问题。

**中层循环（每个剧情章节）**

```
以陌生人身份进入 → 通过解谜积累材料和信息
→ 触发关键道具制作 → 赠予NPC获得认可
→ 解锁更深层的故事 → 进入下一章
```

**外层循环（整个游戏45分钟）**

```
外来陌生人 → 被接纳的倾听者 → 故事的参与者
→ 面对无法改变的遗憾 → 带着遗憾和理解离开
```

### 1.6 爽点节奏规划

| 时间点 | 爽点类型 | 具体内容 |
|--------|----------|----------|
| 第3分钟 | 好奇钩子 | 格桑说「旦增不喜欢问经堂的人」——第一个悬念 |
| 第8分钟 | 信息爽点 | 旦增说「你祖父懂得先听」——第一个认可 |
| 第15分钟 | 信息爽点 | 白玛的猎人故事——隐喻被理解的瞬间 |
| 第20分钟 | 创造爽点 | 残卷完整拼凑，开始制作唐卡 |
| 第28分钟 | 认可爽点 | 赠予道具，NPC第一次真正开口 |
| 第33分钟 | 视觉+叙事冲击 | 进入经堂，空的画框 |
| 第38分钟 | 情感高潮 | 旦增说出真相 |
| 第45分钟 | 情感余韵 | 结局——遗憾但向前 |

### 1.7 NPC无声反馈系统（戴夫式）

玩家完成唐卡复刻并放入经堂画框后，此后每次路过经堂，都能看到NPC在画前的状态变化，**无任何文字提示**：

| 次序 | 内容 |
|------|------|
| 第一次路过 | 格桑站在画前看了很久，没有说话，走了 |
| 第二次路过 | 白玛在画前念了一段经，然后鞠了一躬 |
| 第三次路过 | 旦增第一次走进了经堂，站在画前，背对着玩家 |

*这些不是任务，没有提示。玩家路过时自然看见。藏族人表达情感靠行动，不靠言语。*

## 2. 世界观与背景

### 2.1 地点设定

**措钦村**。藏语意为"大湖旁的村子"，位于拉萨周边，海拔约3000米。

三十余户人家，背靠雪山，面朝一片已干涸的湖床。村口矗立一座玛尼堆，刻有六字真言，旁边散落风化的牛骨。村里经幡不多，主要集中在山顶和经堂屋顶。村中央有一座旧经堂，门上挂铁锁，已关闭十二年。

### 2.2 地区文化还原要求

本游戏场景基于拉萨周边村落（**严禁混入那曲牧区的视觉元素**）：

| 元素 | 准确还原要求 |
|------|-------------|
| **房屋建筑** | 石头或土砖建造，低矮厚重，无钢筋水泥 |
| **村内动物** | 家养奶牛为主（**非牦牛**），藏獒看门 |
| **经幡分布** | 村内极少，主要在山顶和寺庙屋顶；五色顺序严格：蓝（最上）→白→红→绿→黄（最下） |
| **玛尼堆** | 刻六字真言，旁有风化牦牛骨头 |
| **转经筒** | 村口或寺庙旁，生锈但仍有人使用 |
| **村民日常** | 日常穿休闲服，节庆才穿藏装；老人独处时手持佛珠或转经筒，口念六字真言 |
| **经堂外观** | 十分破旧，岁月侵蚀感强，旁边有玛尼石堆、牛骨、少量经幡 |

### 2.3 时代背景

2020年前后。年轻人大多外出务工，留下老人、孩子和一些说不清为何留下的中年人。一家成都文化公司正洽谈将村子开发为民俗旅游点，村民意见不一。而谁都没说出口的，是经堂里的那件事。

### 2.4 唐卡的历史（核心背景）

二十年前，旦增·洛桑的妻子因病去世。按藏族习俗，请僧人算命后，家中供奉白度母唐卡作为亡者的护法图——在藏族文化里，这幅画代替照片，是对逝者最重要的祭奠。

旦增用妻子的白色氆氇（藏族羊毛织物）磨成颜料中的骨白，历时四十九天，亲手绘成一幅《度母护法图》，供奉于村中经堂。这是他这辈子最后一幅唐卡。

十二年前，唐卡失踪，经堂当日关闭。没有报案，没有追查，全村集体沉默。

### 2.5 真相全貌（仅供开发团队）

旦增的儿子——卓玛的哥哥，格桑的父亲——因欠债，趁夜盗走唐卡卖给过路收藏商。旦增发现时已无法追回。他没有揭发儿子，选择用沉默承担，关闭经堂，从此不再作画，不再收徒。

格桑不知道父亲做过这件事，以为父亲只是离家务工。唐卡目前辗转至成都一位私人收藏家手中，仁青曾见过照片但不知完整来龙去脉。

> **故事的真正主题：** 不是失落的唐卡，而是失落的父子关系。不是文化的消逝，而是一个父亲用沉默承担罪与爱的方式。旦增之所以不说，不是因为他在保护自己，而是因为他在保护格桑——不让这个孩子知道父亲是怎样的人。

## 3. 人物设定

### 3.1 玩家角色

| 项目 | 内容 |
|------|------|
| **身份** | 城市长大的年轻人，祖父是已故藏学研究者（三个月前去世，留下一箱田野笔记） |
| **来村原因** | 祖父遗物中有田野笔记，记载了这幅唐卡，附注写着「措钦，问旦增」 |
| **性格预设** | 无固定性格，由玩家对话风格决定 |
| **情感动线** | 从「完成任务」到「真正理解」。游戏结束时，找没找到唐卡并不重要——重要的是理解了为什么它消失了 |

### 3.2 旦增·洛桑（70岁，前唐卡画师）

| 项目 | 内容 |
|------|------|
| **外貌** | 干瘦，手指关节变形，右手拇指有褪色蓝色墨迹，独处时手持佛珠念六字真言 |
| **服装** | 传统藏装，节庆时有盛装 |
| **性格** | 沉默有重量，会回答问题但永远少说一半 |
| **知道什么** | 他是唯一知道完整真相的人，但认为「说出来会伤人，沉默是慈悲」 |
| **AI对话风格** | 用藏族俗语和自然隐喻，句子短，留白多；直接追问唐卡或经堂只会得到沉默，需先聊绘画、聊山才能让他开口；当玩家展示出对唐卡绘制技艺的了解时，解锁更深层信息 |
| **绝不主动说** | 儿子盗画的事。需要玩家走到第四章经堂场景才会触发 |
| **情感核心** | 他的沉默不是为了自己，是为了格桑 |

### 3.3 卓玛（38岁，旦增女儿，村委会副主任）

| 项目 | 内容 |
|------|------|
| **外貌** | 短发，日常穿冲锋衣，手机不离手；仪式场景换藏装 |
| **性格** | 务实、直接、有些疲惫，一只脚在传统里，一只脚在现代里 |
| **知道什么** | 知道唐卡失踪的部分经过，但选择不追究，认为村子未来比一幅画重要；她自己也有盲区 |
| **AI对话风格** | 最容易建立信任的NPC，不需要文化门槛，只需玩家表现出真诚；信息碎片化，AI不应让她表现得比实际知道的更多 |
| **持有物** | 经堂钥匙（与白玛共同持有） |

### 3.4 格桑（19岁，旦增孙子）

| 项目 | 内容 |
|------|------|
| **外貌** | 宽沿帽，背画夹，偏传统村落服装但有年轻人的随意感 |
| **性格** | 敏感好奇，有不安分的能量，崇拜旦增但旦增不肯教他 |
| **知道什么** | 不知道父亲盗画之事；转经筒的动作漫不经心，信仰是继承而非虔诚；无意发现过压在石头下的碎布 |
| **AI对话风格** | 主动找玩家说话，是信息催化剂，但提供的线索有时指向错误方向；对父亲话题敏感，会快速转移话题；温和持续追问会说出五岁时父亲离家的事；会记得玩家之前说过的话并在之后自然引用 |
| **情感核心** | 他是旦增沉默的代价，也是旦增沉默的原因 |

### 3.5 白玛（65岁，半僧人）

| 项目 | 内容 |
|------|------|
| **外貌** | **衣着和普通村民完全一样，不穿僧袍**（重要文化准确性细节，AI描述时应体现长年不吃肉、独处时不停念经、主动管理保护旧经堂，而不是通过服装体现身份） |
| **性格** | 表面随和内里固执，信仰是藏族民间信仰——佛教+本教+他自己的一套逻辑 |
| **知道什么** | 知道经堂为何关闭，但只用「因果」和「业报」解释，不用世俗语言；持有经堂钥匙 |
| **AI对话风格** | 每句话有字面意思和隐藏意思两层，需要理解藏传佛教基本概念才能读懂；玩家引用藏传佛教概念时会显示出真正认可；在仪式重现系统中犯错时，以纠正的方式说出平时不会主动说的内容 |

### 3.6 仁青（45岁，成都文化公司代表）

| 项目 | 内容 |
|------|------|
| **外貌** | 藏族本地人，在成都生活二十年，说话带成都口音，名片印「文化遗产活化专家」 |
| **性格** | 真诚但立场复杂，真的相信旅游开发能保护文化，但方式让村民不舒服 |
| **知道什么** | 通过私人渠道见过唐卡照片，知道它现在在成都，但不知完整来龙去脉 |
| **AI对话风格** | 最主动提供信息的NPC，热情、专业、有点推销感；他说的内容是真实的，但他的存在本身是问题的一部分；是最大的灰色人物 |

### 3.7 藏獒（无名）——情感隐喻角色

旦增家门口的大型藏獒，不是可对话角色，是**全游戏最重要的情感温度计**。

| 进度 | 行为 |
|------|------|
| 第一章 | 冲着玩家狂吠，旦增没有出来制止 |
| 第二章 | 还是吠，但停得稍早一些 |
| 第三章 | 不再吠，只是盯着玩家 |
| 第四章 | 主动走过来，在玩家脚边蹲下 |
| 结局 | 跟玩家走到村口，停下，看着玩家走远 |

*不需要任何文字说明。它是旦增对玩家态度变化的动物镜像。*

## 4. 完整剧情

剧情共分五章，每章对应一个玩家状态的转变。**解谜是推进剧情的唯一手段；AI对话是理解剧情的辅助手段。**

### 第一章·抵达（约8分钟）

**玩家状态：** 完全的陌生人，什么都不知道。

**开场场景（村口）**

画面：静态插画，傍晚时分的措钦村村口。玛尼堆、经幡、远处的雪山。风声，奶牛叫声，藏獒在远处吠叫。无任何教程提示。

唯一可交互物件：玛尼堆旁压着一块石头，石头下有一张褪色纸条，藏汉双文：「旦增·洛桑，措钦村。」

玩家唯一目标：找到旦增。

**首次遭遇：格桑**

格桑在村口画素描，主动开口询问玩家来意。这是游戏第一次AI对话，格桑会告诉玩家旦增在哪里，但无意说出：「旦增爷爷不喜欢外地人，特别是问经堂的那种。」——这是第一个悬念。

**首次遭遇：藏獒与旦增**

玩家走向旦增院子，藏獒狂吠，旦增没有出来。玩家推开虚掩的院门，旦增坐在院中，手持佛珠，嘴唇微动，晨光打在他变形的手上。

旦增看见玩家，沉默片刻，说：「你是那个研究者的孙子。」然后转身回屋。没有说请进，也没有说走开。

章节结束。玩家获得第一个问题：**他怎么知道我是谁？**

### 第二章·试探（约10分钟）

**玩家状态：** 开始接触，遭遇第一堵墙。

**可探索区域开放**

玩家可自由在村子里走动，与卓玛、白玛、仁青进行初次AI对话，收集表层信息：

- **卓玛：** 听到唐卡就沉默两秒，然后说「那幅画没了，已经很久了」，不再多说。
- **白玛：** 「唐卡？画是画，心是心，丢的不是画。」玩家无法理解，但会记住。
- **仁青：** 主动提起唐卡，说自己见过照片，愿意详谈，但玩家会感觉他的热情里有其他东西。

**第一堵墙：旦增的沉默**

玩家回去见旦增，无论怎么问唐卡，旦增只有两种回答：「不知道」，或沉默。

卓玛此前无意说过一句话：「父亲唯一愿意开口的，是聊他年轻时画过的那些山。」

**突破口：谈画，不谈唐卡**

玩家转换话题，问旦增绘画本身——颜料怎么调、唐卡的线条规矩、藏区的山是什么颜色——旦增开始说话。

这是游戏核心机制的首次完整呈现：**先成为一个愿意倾听的人，才有资格成为追问的人。**

旦增在这次对话结束时说：「你祖父当年也是这样问我的。他懂得先听，再问。」

章节情感爽点——第一个认可信号。

### 第三章·裂缝（约12分钟）

**玩家状态：** 核心矛盾浮出水面，开始拼凑真相。

**事件：旅游开发会议**

仁青召集非正式会议。卓玛和白玛出现明显分歧：

- 卓玛支持开发，条件是「经堂不能动」
- 白玛反对，原因含糊：「根已经动了，这是果报」

现场气氛骤凝。玩家注意到旦增（在旁听）转身离开。**这是第一个明确的「有事隐瞒」信号。**

**格桑的无意线索**

格桑情绪不对。他说整理旦增院子时翻到一块压着什么的石头，被旦增拿走了那块布，什么都没解释。

格桑说父亲的事：「我爸在我五岁前就走了，我爷爷说他去远方了，但我知道他没死，只是不回来。」——埋下情感线的关键对话。

白玛讲了一个藏族故事：一个猎人射伤了神山上的鹿，鹿没有死，但猎人的手从此再也不能拉弓。不是诅咒，是业报。

玩家问：「您说的猎人是谁？」

白玛：「说的是故事，不是人。但这个村子里，有人的手不能再画画了。」

**章节结束**

玩家直接问旦增：「您为什么不再画唐卡了？」

旦增看着自己变形的手，沉默很久，然后说：「手老了。」

玩家知道这不是真相，但也知道现在还不是时候。

### 第四章·经堂（约10分钟）

**玩家状态：** 触碰核心秘密，迎来最大情感转折。

**进入经堂的条件**

经堂钥匙由卓玛和白玛共同持有。进入条件：完成唐卡复刻并赠予其中一人（见5.4节）。卓玛或白玛其中一人被打动后，主动提出带玩家进入经堂。

**经堂内部**

正面墙上一个空的画框。四周墙壁残留褪色壁画。地上翻倒的酥油灯，已干涸。

「格桑不知道。我希望他永远不知道。」

## 5. 玩法系统

*解谜驱动主线。所有玩法设计遵循一个原则：文化知识是谜题的钥匙，不是学习材料。*

### 5.1 场景与交互系统

#### 场景节点清单

```
village_entrance    村口·玛尼堆
village_square      村子中央广场
dangzeng_courtyard  旦增家院子
dangzeng_bedroom    旦增卧室（第四章后解锁）
gesang_corner       格桑的角落
temple_exterior     经堂外
temple_interior     经堂内部（第四章解锁）
mountain_peak       山顶·经幡
sangtai             煨桑台
```

#### 场景数据结构

```json
{
  "sceneId": "dangzeng_courtyard",
  "background": "图片资源路径",
  "visibleFrom": ["village_square"],
  "exits": [
    { "target": "village_square", "position": { "x": 0, "y": 0 } }
  ],
  "interactables": ["物件ID数组"],
  "npcsPresent": ["NPC ID数组，含条件"],
  "stateVariants": {
    "chapter1": {},
    "chapter3": {}
  }
}
```

#### 可交互物件数据结构

```json
{
  "itemId": "mani_stack_note",
  "sceneId": "village_entrance",
  "hitArea": { "x": 0, "y": 0, "width": 100, "height": 100 },
  "icon": "hover时是否高亮提示",
  "unlockCondition": null,
  "onInteract": "triggerEventID",
  "oneTimeOnly": true,
  "resultType": "dialogue | item | scene_change"
}
```

#### 软性开放世界规则（核心机制）

**场景物理上全部开放，但内容根据玩家进度动态变化。** 玩家始终能走到任何地方，但「能看到什么、能做什么」由进度决定。

实现方式：每个场景的 `interactables` 和 `npcsPresent` 列表根据全局进度变量动态过滤。

**不要用「锁定/未锁定」二元状态**，而是用「内容图层」叠加——同一个物件在不同进度下，`onInteract` 指向不同的对话或反应内容，物件本身始终可点击。玩家永远不会遇到「灰色不可点」的挫败感。

示例——经堂外场景在不同进度下的差异：

| 进度阶段 | 可见物件 | NPC反应 |
|----------|----------|---------|
| 第一章 | 锁着的门（可点击但只触发对话） | 白玛：「这扇门关了很久了」 |
| 第三章 | 门+新增的玛尼石刻痕 | 卓玛开始愿意谈论钥匙的事 |
| 第四章 | 门自动变为可进入状态 | 无需额外点击确认，直接转场 |

### 5.2 NPC信任系统

#### 设计原则

没有可见的信任值数字。内部必须有数值，但UI层绝不能显示数字或进度条。信任通过NPC行为变化体现：从主动转身离开，到停下来多说一句，到主动来找玩家，到在某个重要时刻说出关键的话。

#### 信任值数据结构

```json
{
  "npcId": "danzeng",
  "trustLevel": 0,
  "trustStage": "stranger",
  "flags": {
    "hasDiscussedPainting": false,
    "hasSeenBrokenCloth": false,
    "hasGivenThangka": false
  },
  "unlockedDialogueTopics": []
}
```

#### 信任阶段映射表（以旦增为例）

| 数值区间 | 阶段标签 | 行为表现 | 解锁内容 |
|----------|----------|----------|----------|
| 0–20 | `stranger` | 只回答「不知道」或沉默 | 无 |
| 21–45 | `listener` | 愿意聊绘画、聊山，不聊唐卡 | 残卷第四页线索 |
| 46–70 | `acknowledged` | 主动提起祖父，认可玩家 | 深层对话选项 |
| 71–100 | `confidant` | 主动出现，说出真相 | 经堂剧情触发 |

每个NPC需要策划单独定义阶段映射（各NPC阶段名称相同，但触发行为和解锁内容不同）。

#### 信任值增长规则

信任值靠「对话质量」而非「对话次数」简单累加：

| 触发器 | 增量 | 说明 |
|--------|------|------|
| 话题契合 | +3~5 | 玩家主动聊NPC感兴趣的话题，由AI返回语义标签触发 |
| 解谜成果展示 | +8~15 | 完成某谜题后向NPC展示成果 |
| 文化理解正确 | +5 | AI对话中表现出对藏族文化的准确理解 |
| 触碰禁忌话题 | -5~10 | 信任不足时直接追问核心秘密 |
| 长时间不互动 | 不衰减 | 信任只增不减，保护玩家探索的安全感 |

#### AI返回标记与信任系统的对接接口

AI对话每次返回时，除对话文本外，还需返回结构化标记对象供前端更新信任值：

```json
{
  "dialogueText": "你祖父当年也是这样问我的...",
  "trustDelta": 8,
  "triggeredFlags": ["hasDiscussedPainting"],
  "unlockedTopics": ["祖父的田野调查"],
  "isKeyLine": false
}
```

实现建议：在每个NPC的System Prompt末尾追加格式要求，让AI在对话正文后附加此JSON；前端解析后更新本地状态，正文部分展示给玩家，JSON部分不展示。

> 🔧 **实装批注（2026-07-02）｜AI↔信任对接已实装，但采用了不同于本节设计的方案。**
> 本节设想"每轮回复内联附带 JSON 标记"，实装改为**事后裁判**（`apiClient.js` 新增 `judgeAiChat()`）：自由对话结束（对话框关闭）时，把完整聊天记录发一次非流式请求给 LLM 评估，按"聊绘画/山/颜色/倾听 → +1~+5；逼问唐卡/隐私/冒犯 → -1~-5"打分，解析 JSON 后调 `onAdjustTrust`。
> 改方案的理由：①内联 JSON 会污染流式输出（正文和标记混流，截断即坏）；②裁判独立于角色扮演 prompt，角色不必"自己给自己打分"，更难被玩家话术操纵；③每次自由对话只多一个短请求，成本可控。
> 防刷设计：每个 NPC 的 AI 正向收益累计上限 15（localStorage `shanglu_ai_trust_gained_*`），负向不设限。裁判失败静默忽略。`triggeredFlags / unlockedTopics` 部分**未实装**（flag 仍全部由固定剧情节点触发），为后续增强点。

### 5.3 解谜系统

#### 5.3.1 残卷收集系统

玩家在场景中发现一本《唐卡绘制古法》残卷（祖父田野笔记的一部分），记载了《度母护法图》的绘制方法，但残缺分散。

**残卷数据结构**

```json
{
  "scrollPageId": "scroll_page_1",
  "location": "village_entrance.mani_stack",
  "unlockCondition": "always_visible_but_findable",
  "contentRevealed": "构图基础比例规则",
  "collected": false
}
```

**四页残卷触发条件一览**

| 残卷 | 位置 | 解锁条件 | 内容 |
|------|------|----------|------|
| 第一页 | 玛尼堆石缝 | 无条件，仔细检视即得 | 构图比例基础 |
| 第二页 | 白玛念经角落（哈达下面） | 白玛信任值≥21（listener阶段） | 佛青颜料隐喻线索 |
| 第三页 | 格桑画夹（格桑不自知） | 格桑主动展示画夹的对话分支触发后 | 藏金颜料制法线索 |
| 第四页 | 旦增院子石头下 | 旦增信任值≥21且已完成「谈画」对话 | 骨白颜料情感线索 |

残卷的分布本身是一张社交地图——每一页都和一个NPC的信任程度挂钩。玩家可能拿到残卷的顺序因人而异（有意为之），**唐卡制作界面必须能正确处理「残卷不全也可先进入，只看已解锁部分」的情况**。

#### 5.3.2 颜料采集与制作系统

**五种颜料数据结构**

```json
{
  "pigmentId": "fojing_blue",
  "name": "佛青",
  "culturalMeaning": "智慧与虚空",
  "rawMaterialFound": false,
  "recipeUnderstood": false,
  "craftable": false,
  "craftedCount": 0
}
```

**五种颜料总览**

| 颜料 | 文化含义 | 原材料线索 |
|------|----------|-----------|
| 佛青 | 智慧与虚空 | 产自高原矿石，色如深秋夜空 |
| 朱砂红 | 生命力与护法 | 来自赤铁矿，需火煅烧 |
| 藏金 | 光明与佛性 | 植物汁液与矿粉混合 |
| 松石绿 | 自然与慈悲 | 高原特有植物的茎叶 |
| 骨白 | 纯洁与涅槃 | 来历成谜，残卷语焉不详 |

**原材料采集谜题（观察判断型）**

以佛青矿石为例：

- 场景中放置两个视觉相似的蓝色矿石：`blue_stone_correct` 和 `blue_stone_decoy`
- 两者初始外观相同，定义「光照角度」互动——特定角度下，正确矿石显现「深秋夜空」般渐变色，错误的保持纯色
- 点击正确矿石触发采集成功；点击错误矿石不惩罚，只返回NPC旁白提示「这个颜色不太对」

此模板可复用于松石绿等其他需要场景观察的颜料，只需替换判断条件和视觉资源。

**颜料制法解锁——各颜料渠道**

| 颜料 | 制法来源类型 | 触发方式 | 前端对接点 |
|------|------------|---------|-----------|
| 佛青 | AI对话隐喻 | 白玛对话中AI返回特定flag | 监听 `triggeredFlags` 中是否含 `recipe_fojing` |
| 朱砂红 | 场景细节 | 旦增院子铜炉刻字，仅在特定光照/缩放下可读 | 场景物件多阶段交互（点击→提示→擦拭动作→显示刻字） |
| 藏金 | 对话+残卷联动 | 格桑对话提及植物，需与已收集残卷比对 | 前端做简单「比对」小游戏或自动判断已有信息是否匹配 |
| 松石绿 | 场景物件 | 点击老药方物件，对照已知植物特征 | 文本比对型谜题，「选择正确植物」交互 |
| 骨白 | 情感核心谜题 | 见下方独立流程 | 单独线性引导逻辑 |

**骨白颜料——情感核心谜题（不可复用通用模板）**

骨白是全游戏唯一不能仅靠「采集+制法」两步完成的颜料，触发链如下：

1. **前置条件：** 玩家已通过对话理解骨白与旦增亡妻的关联（AI对话触发flag: `understood_bone_white_meaning`）
2. **玩家向旦增追问骨白制法**（专门设计固定对话入口，在特定信任阶段出现的固定对话选项，非自由聊天）
3. **旦增交付氆氇残片**（白玛保存的同款碎片）作为道具，前端在道具栏新增 `bone_white_cloth` 物件
4. **玩家在颜料制作界面「研磨」该物件**，触发骨白解锁动画与旁白

残卷最后记载：「此色不可强求，需以至亲之念研磨，色方正。」

> **骨白谜题的意义：** 玩家在找颜料时，理解了那幅唐卡对旦增的含义——不只是文物，而是妻子的遗像。这一段必须保证每个玩家都不会错过，需单独写线性引导逻辑。

#### 5.3.3 六字真言拼图

六个石块乱序放置在经堂门口场景，玩家需要拖拽排列成正确顺序（唵嘛呢叭咪吽）。

```json
{
  "correctOrder": ["om", "ma", "ni", "pad", "me", "hum"],
  "currentOrder": [],
  "hintSource": "白玛此前对话中念出过完整顺序",
  "checkOnEachMove": true
}
```

**设计意图：** 这个谜题考验的是玩家有没有认真听白玛说话，不是记忆力游戏。游戏内需提供「对话回顾」入口（见7.2节），让错过台词的玩家可以查阅。

> 🔧 **实装批注（2026-07-02）**：解开六字真言拼图新增信任奖励——在白玛面前排对真言，白玛 +12（一次性）。这是白玛线从"陌生人"进入"倾听者"的主要台阶之一（初见+10 与拼图+12 合计 22 > 21）。

#### 5.3.4 经幡排序谜题

**设计定位：** 经幡谜题是白玛专属的剧情台阶，在六字真言拼图之前给白玛线索增加一个递进节点。两个谜题共用同一套拖拽排序模板（`DragSortPuzzle` 组件），形成「低信任→经幡谜题，高信任→六字真言拼图」的递进关系。

**触发条件：** 白玛信任值达到 `listener` 阶段（21–45区间）后，白玛对话池解锁新台词，同时山顶场景中经幡从纯展示物件变为可交互谜题物件。

提示台词示例：「风把颜色吹乱了，蓝天白云火焰绿水大地，谁还记得它们该在哪里。」

**谜题数据结构**

```json
{
  "correctOrder": ["blue", "white", "red", "green", "yellow"],
  "currentOrder": [],
  "unlockCondition": "baima.trustStage >= listener",
  "checkOnEachMove": true,
  "solved": false
}
```

**完成反馈：** 触发白玛专属对话，将经幡的「颜色归位」和经堂的「门关闭」做隐喻类比：「色归其位，心才安。这经堂的门，比这经幡乱得更久。」

完成后给予白玛信任值 +10，与旦增、卓玛测试面板量级保持一致。

> 🔧 **实装批注（2026-07-02）**：两处修正——①经幡+10 改为 `awardTrustOnce` 一次性授予（原实现配合"每次进山顶自动重置拼图完成标记"的遗留测试代码，可无限重复加分；该测试代码已删除，拼图完成状态现在正常持久化）。②**答案泄漏修复**：广场经幡检视文本（`dlg_prayer_flags`）原来直接写出"蓝是天，白是云，红是火，绿是水，黄是土"，等于把谜底贴在谜题旁边；已改为只说"五色各有所属"，答案来源收敛到白玛的提示台词（需信任达到倾听者），符合"知识做门槛"原则。

**与六字真言拼图的对比**

| 项目 | 经幡排序 | 六字真言拼图 |
|------|----------|-------------|
| 触发信任阶段 | listener（21分以上） | 不限定，固定出现在经堂门口 |
| 场景位置 | 山顶·经幡 | 经堂外门口 |
| 提示来源 | 白玛新增台词 | 白玛此前对话中念出的完整顺序 |
| 完成后效果 | 信任值+10，解锁隐喻台词 | 解锁进入经堂的最后一道门槛 |
| 谜题模板 | `DragSortPuzzle`（复用） | `DragSortPuzzle`（原始实现） |

#### 5.3.5 图像解读系统（唐卡记忆画布）

玩家在游戏过程中收集唐卡局部照片碎片，在「记忆画布」界面将其拼合，每个区域被点击时触发对应解读。

**设计意图：** 让玩家通过追查一幅已经不存在的画，在心里重建了它。

| 区域 | 解锁前置 | 交互方式 | 结果 |
|------|----------|----------|------|
| 区域一·度母手势（右手持莲，左手施无畏印） | 获得仁青提供的对应碎片 | 点击后弹出解读文本+与旦增行为的呼应提示 | 文化知识+人物呼应 |
| 区域二·背景山形（与措钦村雪山轮廓一致） | 场景对比 | 并排显示场景实拍与唐卡局部，玩家手动确认「相似」 | 纯视觉发现，无对错判定 |
| 区域三·度母左眼（微睁，看见真相但不说） | 积累信息量达阈值 | 自动解锁，不需操作 | 核心象征揭示，文本展示 |
| 区域四·角落藏文（含旦增亡妻名字的一部分） | 请白玛辨认（专属对话触发） | 固定剧情对话，非自由聊天 | 名字关联揭示 |
| 区域五·画面空白（度母胸前镜子，藏族文化里镜子照见真相） | 完成其余四个区域 | 自动触发 | 故事核心隐喻收束 |

**技术要点：** 区域五需要前四个全部完成才能触发，这是唯一的强制顺序点，需在代码里做完成度检查。

#### 5.3.6 仪式重现系统（煨桑台）

村里有一座废弃的煨桑台，玩家可以重新点燃它。

**核心设计：** 正确不是操作正确，而是文化意义上的正确。

**四个子条件**

| 条件 | 正确选项来源 | 错误反馈 | 数据结构字段 |
|------|------------|---------|-------------|
| 植物种类 | 场景采集，松柏/杜松等可选项 | 白玛皱眉不语（无文本，纯演出） | `plantType: "correct" \| "incorrect"` |
| 燃烧顺序 | 残卷或白玛对话提及「先净后请」 | 烟雾视觉效果显示往下走 | `burnOrder: array`（判断顺序） |
| 诵念内容 | 白玛闲聊时出现过的词句 | 白玛停下纠正，触发额外剧情 | `chantText: string`（字符串匹配） |
| 点火时机 | 白玛提到的藏历时辰 | 无强制错误反馈，仅做氛围彩蛋 | `timing: optional`（非强制判定） |

> **说明：** 点火时机不建议做成强制判定——藏历时辰实时换算对开发成本过高，且容易刻板。建议设为「氛围彩蛋」：凑巧在游戏内时间对应时段点火，触发一句特别旁白，但不影响仪式成败。

**仪式成功演出（固定时间轴序列，不可跳过）**

1. 烟雾笔直上升的动画
2. 风声渐弱
3. 村民NPC陆续从各自场景「走入」煨桑台场景（B实现简单的NPC位置切换逻辑）
4. 旦增最后出现——无人说话，但玩家感觉到村子里某种东西松动了

> 🔧 **实装批注（2026-07-02）**：仪式成功现在是**全村信任跃迁点**——白玛 +15、旦增 +15、卓玛 +12（各一次性）。其中旦增 +15 是刻意的关键设计：它把旦增推过知己门槛（71），解锁骨白秘密，从而解开"骨白需知己→知己需赠唐卡→唐卡需骨白"的死锁。演出文本"旦增第一次走出那个院子"正是这次数值跃迁的叙事化身——机制与情感同拍。

### 5.4 唐卡制作系统

颜料集齐后，进入制作界面。不是自由绘画，而是**图案还原谜题**，分三步骤Modal界面实现。

#### Step 1：构图谜题（网格化拖拽）

玩家将度母身体部位元素拖拽到符合「藏族量度规范」比例的网格位置。

```json
{
  "gridRules": {
    "headPosition": { "row": "1-2", "col": "4-6" },
    "torsoPosition": { "row": "3-6", "col": "3-7" },
    "tolerance": 1
  }
}
```

判定逻辑：每个元素拖放后比对网格坐标是否落在容差范围内，全部正确才能进入下一步；错误时元素自动弹回原位，显示来自残卷的提示文本（不用错误音效，避免挫败感）。

#### Step 2：颜色谜题

玩家从已制作好的颜料中，为构图的不同区域分配颜色。

错误颜色不是直接「错误」提示，而是触发白玛的旁白解释为什么错——**把谜题反馈转化为文化学习的关键设计点**。需为每个可能的错误分配预设解释文本ID（由A提供文案）。

#### Step 3：细节谜题——度母左眼

不是操作型谜题，而是**选择型**——界面呈现几种「心中所念」的隐喻选项（不直接写「妻子」），玩家需根据此前所有对话积累的理解选择最贴近的一项。

技术实现：允许「选错」，选错不会卡住流程，但旦增后续的反应文本会有细微差异（通过flag区分），不惩罚探索。

#### 制作完成的产出物

```json
{
  "thangka_replica": {
    "completed": true,
    "step1Choices": {},
    "step2Choices": {},
    "step3Choice": "选择ID",
    "quality": "深刻理解 | 基础理解",
    "completedAt": 1234567890
  }
}
```

制作完成的唐卡记录玩家在三个步骤中的具体选择——**用于后续赠予分叉判断玩家的理解程度，但不影响是否能赠予**（赠予本身永远可行，差异体现在NPC反应文本上）。

#### 赠予分叉系统

同一件道具给不同NPC，触发不同的故事层。这是本游戏重玩价值的核心来源。

**通用赠予交互流程**

1. 玩家在道具栏选中 `thangka_replica`，点击任意已解锁的可赠予NPC
2. 系统检查该NPC是否已被赠予过（`giftedNPCs` 数组判重）
3. 触发该NPC专属的赠予反应对话（**固定剧情，非AI自由生成**——关键节点必须保证叙事准确）
4. 赠予后更新对应flag，解锁后续场景或对话分支

**重要设计澄清：** 赠予动作本身可以重复执行给不同NPC（旦增、格桑、白玛都可以分别赠予一次），系统需要支持「一份道具，多次使用」，而不是消耗品逻辑。

**三NPC赠予反应对比**

| 赠予对象 | NPC反应 | 触发内容 |
|---------|---------|---------|
| **旦增** | 他用手指轻触骨白，说「你知道这个白色是怎么来的」，打开卧室里从未开过的木箱，里面是妻子的照片和氆氇残余 | 绘画线索与情感核心 |
| **格桑** | 格桑问「你怎么知道比例的」，沉默后说「爷爷从来不教我，我以为是我不够好」，翻出自己画过的错误比例度母素描 | 家族线索，祖孙关系裂缝松动 |
| **白玛** | 他第一次用直白语言说话，不用隐喻：「画它的人把命画进去了，拿走它的人不知道自己拿走了什么。」 | 宗教线索，唐卡失踪的道德重量 |

### 5.5 卡关提示机制

#### 设计原则

提示给的不是答案，而是文化语境的线索。提示同时推进文化理解和人物关系，不能是纯粹的「功能性攻略提示」。

#### 卡关判定规则

采用时间+尝试次数双重判定：

| 维度 | 阈值 | 触发行为 |
|------|------|---------|
| 时间阈值 | 同一谜题界面停留超过3分钟未推进 | 相关NPC可点击状态出现细微高光，鼓励玩家主动去问 |
| 尝试阈值 | 同一交互点重复触发3次以上仍未成功 | 同上 |

**重要原则：** 不要做成自动弹窗。这款游戏要求玩家始终感觉「自己在探索」，正确做法是让提示的获取也需要玩家主动行为（点击NPC对话），只是降低这次对话触发提示的概率门槛。

#### 各谜题对应提示设计

| 卡关谜题 | 触发NPC | 提示文本方向 | 技术实现 |
|----------|---------|------------|---------|
| 骨白材料 | 白玛 | 闲聊中提到「旦增老伴走的时候，他一件衣服都没留，只留了一块布」 | 在白玛对话池中加入低权重但条件触发的台词 |
| 六字真言顺序 | 格桑 | 「白玛每天念的那几个字，我从小就会背」并直接念出 | 格桑对话分支，触发后直接显示正确顺序文本 |
| 佛青矿石辨认 | 卓玛或场景内 | 提示玩家尝试改变光照/视角 | UI层提示性动画，非文字 |
| 构图比例 | 旦增 | 重新强调「量度」概念的对话 | 旦增对话池追加 |

### 5.6 难度与节奏曲线

#### 整体难度曲线

45分钟内谜题难度呈「低-中-高-低」，而非持续递增——结局阶段需要情感释放：

| 章节 | 时间区间 | 谜题难度 | 谜题类型 |
|------|----------|----------|---------|
| 第一章 | 0–8分钟 | 极低 | 纯探索，建立操作习惯 |
| 第二章 | 8–18分钟 | 低 | 单步骤对话型谜题（谈画破冰） |
| 第三章 | 18–30分钟 | 中 | 残卷收集+颜料采集（多步骤但容错高） |
| 第四章 | 30–40分钟 | 中高 | 唐卡制作三重谜题（全游戏操作峰值） |
| 第五章 | 40–45分钟 | 无 | 无操作型谜题，纯叙事抉择 |

#### 容错设计统一规范

全游戏没有「失败结局」或「游戏结束」状态。所有谜题的错误反馈遵循以下统一规范：

1. 错误操作不会导致道具丢失或状态倒退
2. 错误反馈优先用演出/旁白形式，其次才是文字提示，最后才考虑明确的「错误」视觉信号（红色高亮等），且应尽量避免
3. 任何谜题理论上允许无限次尝试
4. 错误尝试本身可以成为卡关提示系统的触发条件之一（叠加计数）

## 6. 技术架构

### 6.1 技术栈

| 层级 | 方案 |
|------|------|
| **前端框架** | React（组件化管理场景、对话框、谜题界面，适合多状态切换） |
| **状态管理** | React Context 或轻量状态库（如Zustand），统一管理玩家进度、信任值、道具栏等全局状态 |
| **持久化存储** | 浏览器 localStorage，无需后端服务器 |
| **AI对话接入** | 调用大语言模型API（OpenAI兼容格式），每个NPC独立维护对话上下文 |

### 6.3 AI系统设计

#### NPC Prompt设计框架

每个NPC的System Prompt包含以下五个层次：

1. **身份背景：** 年龄、职业、生活经历、与其他NPC的关系
2. **性格特质：** 说话风格、情绪反应模式、惯用表达方式
3. **知识边界：** 明确知道什么，明确不知道什么
4. **秘密分级：** 公开信息 / 需要信任才说 / 永远不主动说
5. **文化语言规范：** 如何体现角色的文化背景和身份

#### 各NPC的AI设计要点

**旦增·洛桑**
- 说话方式：用藏族俗语和自然隐喻，句子短，留白多
- 触发规则：直接追问唐卡或经堂→沉默或「不知道」；聊绘画、聊山、聊颜色→逐渐开口
- 文化门槛：当玩家展示出对唐卡绘制技艺的了解时，解锁更深层信息
- 绝不主动说：儿子盗画的事

**白玛**
- 说话方式：隐喻体系，每句话有字面和隐藏两层含义
- 身份识别提示：体现长年不吃肉、管理寺庙，而不是通过服装（不穿僧袍）
- 文化门槛：玩家引用藏传佛教概念时会显示出真正认可
- 特殊规则：仪式重现系统中犯错时，以纠正的方式说出平时不会主动说的内容

**格桑**
- 说话方式：直接、热情，有时鲁莽，提供的线索不一定准确
- 特殊规则：对父亲话题敏感，会快速转移话题；温和持续追问才会说出五岁时父亲离家的事
- 记忆机制：记得玩家之前说过的话，并在之后自然引用

**卓玛**
- 说话方式：直接务实，偶尔疲惫，用现代词汇
- 最容易建立信任，不需要文化门槛
- 信息碎片化，她自己也有盲区，AI不应让她表现得比实际知道的更多

**仁青**
- 说话方式：热情、专业、有点推销感
- 主动提供信息，但玩家应感觉到他的动机不纯粹
- 灰色设定：他说的内容是真实的，但他的存在本身是问题的一部分

#### AI自由度边界控制

前端层面需对AI对话设置边界控制，不完全依赖System Prompt：

| 控制项 | 实现方式 |
|--------|---------|
| 单轮对话长度 | 前端限制单次玩家输入字符数（建议300字以内） |
| 话题强制纠偏 | 简单关键词黑名单做基础过滤；复杂语义判断交由System Prompt层 |
| 对话轮次软上限 | 单次会话超过约15轮无实质推进时，NPC触发「需要去做点别的事了」类型收尾对话 |

> 🔧 **实装批注（2026-07-02）｜AI 与剧情的双向同步已打通。**
> - **游戏→AI（新增）**：`Scene.jsx` 新增 `getAiContext(npcId)`，在自由对话的 system prompt 末尾注入「当前游戏进度」段——信任阶段、是否通过"先听再问"测试、是否已知骨白来历、是否已赠唐卡、煨桑教学进度、经堂是否已开。解决"AI 旦增在玩家赠完唐卡后仍装作不认识玩家"的时序穿帮。注入段明确标注"角色扮演参考，不要向玩家复述"。
> - **AI→游戏（新增）**：对话裁判 `judgeAiChat()`，详见 §5.2 批注。
> - 本表的三项边界控制（输入长度限制/关键词黑名单/轮次软上限）**均未实装**，为遗留待办。

#### 技术方案：localStorage

无需后端，玩家关闭浏览器后数据保留在本地。需在游戏内说明：更换设备或清除浏览器数据会丢失存档。

#### 持久化数据结构

```json
{
  "saveVersion": "1.0",
  "currentScene": "village_square",
  "playerName": "玩家自定义",
  "npcTrust": {
    "danzeng": { "trustLevel": 0, "trustStage": "stranger", "flags": {}, "unlockedDialogueTopics": [] },
    "baima": {},
    "gesang": {},
    "zhuoma": {},
    "renqing": {}

  "inventory": ["scroll_page_1", "scroll_page_3"],
  "pigments": {

  },

  "dialogueHistory": {
    "danzeng": []
  },

```

#### 保存触发时机（自动存档）

- 每次场景切换时
- 每次对话结束（AI返回一轮完整应答）后
- 每次谜题状态变化（如颜料制作完成）后

技术实现：用一个全局的 `saveState()` 函数，在关键状态变更处统一调用，避免分散写 `localStorage.setItem`。

> 🔧 **实装批注（2026-07-02）｜本轮新增的持久化键（均带 `shanglu_` 前缀，新游戏 `deleteSave()` 自动清除）：**
> - `shanglu_trust_awards`——一次性信任授予的领取记录（key→bool 映射），防止对话重看/谜题重解重复加分。
> - `shanglu_ai_trust_gained_<npcId>`——AI 对话裁判正向收益累计（上限 15/NPC）。
> - `shanglu_ch1_entry_shown`——第一章开场内心OS是否已播。
> - 场景状态新增字段：`hallFrameViewed / hallMuralViewed / hallClothViewed`（经堂三遗物检视追踪）、`ch4TruthDone`（真相线完成标记）。
> - `danzengSceneBCompleted`——旦增"先听再问"测试通过标记（存于 npcTrust flags）。
> - 另：**调试模式开关**——🔧 信任调试面板与 💾 保存布局按钮不再对玩家可见，仅当 URL 带 `?debug` 或 `localStorage.shanglu_debug === '1'` 时显示。

#### 读档与重新开始

1. 存在存档 → 直接恢复到上次场景与状态，无需额外确认弹窗（保持沉浸感）
2. 不存在存档 → 进入新游戏流程

设置菜单中提供「重新开始」选项，点击后**需二次确认**（这是少数允许使用弹窗确认的场景，因为涉及数据清除），确认后清空所有 localStorage 对应字段。

### 7.2 引导与新手体验系统

#### 设计决策：极简隐性引导

总GDD明确要求「无任何教程提示」，但完全没有引导会导致部分玩家不知如何开始交互。隐性引导方案如下：

| 引导目标 | 实现方式 | 触发时机 | 是否可关闭 |
|----------|----------|----------|-----------|
| 物件可点击提示 | 鼠标悬停时轻微的光晕呼吸动画（而非图标或文字） | 玩家进入新场景后，若30秒内无任何点击行为，第一个关键物件开始极轻微高光提示 | 可在设置中关闭 |
| 对话框使用方法 | 首次打开AI对话框时，输入框内有极淡的placeholder文字示例（如「问问他关于唐卡的事…」） | 仅第一次对话触发，之后不再出现 | 自动消失 |
| 场景切换方式 | 场景边缘出口区域，鼠标悬停时光标变化+轻微脚步光效 | 常驻，属于UI反馈 | 不可关闭 |

**原则：** 所有引导必须能被理解为「游戏世界本身的反应」，而不是「系统在教你怎么玩」。光晕、呼吸感、光标变化都符合；文字气泡教程「点击这里开始」不符合，应避免。

### 7.3 反馈与状态提示系统

#### 统一交互反馈规范

| 物件状态 | 视觉表现 | 点击结果 | 示例 |
|---------|---------|---------|------|
| 可交互·未触发 | 静止状态，hover时轻微高亮 | 触发对应事件 | 残卷藏匿点 |
| 可交互·已触发过 | 视觉保持原样，内部标记为已读 | 重复点击显示简短的「已经看过」类回应 | 已读取的场景细节物件 |
| 暂不可交互 | 物理上仍可点击（软性开放原则） | 返回该阶段对应的占位反应文本 | 尚未到时机的经堂门 |
| NPC对话入口 | 立绘呼吸动画，hover时光标变化为对话气泡图标 | 打开AI对话框 | 所有NPC |

#### 谜题正确/错误的统一反馈语言

建议做成两个可复用的React组件，所有谜题统一调用：

- **`CorrectFeedback`：** 轻微金色光效+短促钵声音效（全程不超过5次的预算，需与A、C协调具体使用节点）
- **`GuidanceFeedback`：** 无音效，仅展示一行来自残卷或NPC的旁白文字，样式与对话框统一但出现位置在谜题界面内部

### 7.4 菜单与设置系统

#### 设置菜单功能范围

| 功能 | 说明 |
|------|------|
| 音量调节 | 背景环境音/音效音量独立滑块；无BGM音量项（本游戏无背景音乐，仅两处特殊音乐场景，归类入音效） |
| 重新开始 | 二次确认后清空存档，回到游戏开场 |
| 对话历史回顾 | 见下方 |
| 关于/制作信息 | 团队名单、致谢藏族文化顾问 |

#### 对话历史回顾系统（线索回顾）

专门为「需要记住NPC说过的话」的谜题（如六字真言拼图）设计，避免玩家因没记下台词而过度卡关：

- 从场景内固定UI入口进入（建议放在屏幕角落，不强制使用）
- **分NPC展示关键对话片段，而非完整对话记录**——只摘录被标记为 `isKeyLine: true` 的台词
- 技术实现：在NPC对话返回的结构化数据中增加 `isKeyLine` 字段，由A在设计台词时标注哪些是「今后可能需要回顾的关键信息」

#### 道具栏与图鉴

常驻UI，展示：
- 已收集的残卷页数（如「2/4」——这个数字提示是允许的，是收集型反馈，不是情感数值）
- 已掌握的颜料种类
- 当前是否持有唐卡复刻品

点击任意已收集物品可查看详情卡片（文字+示意图标），不消耗点击次数限制。

## 8. 美术与音效规范

### 8.1 总体视觉风格

静态插画，水彩与矿物颜料质感——呼应唐卡的颜料传统。色调以赭石、藏青、金黄为主，高光用白，**不用纯黑描边**。场景光线随时间变化（早晨冷白、正午强烈、傍晚橙金、夜晚深蓝）但不影响玩法，只影响氛围。

### 8.2 NPC立绘规范

半身立绘，对话时有细微呼吸动画和眼神变化，不是动漫式夸张，而是极微小的真实感。

| NPC | 立绘要点 |
|-----|---------|
| 旦增 | 传统藏装，手持佛珠，右手拇指有褪色蓝墨迹，手指关节变形 |
| 卓玛 | 冲锋衣+手机，特定场景有藏装版本 |
| 格桑 | 宽沿帽，画夹背包，传统但随意 |
| 白玛 | **普通村民衣着，无任何僧人标志**，通过神情和动作体现身份 |
| 仁青 | 现代藏族都市风，名片道具 |

### 8.3 唐卡美术要求

参考老旧唐卡，而非色泽鲜艳的新唐卡。色彩斑驳，几乎难以一眼分辨绿度母还是白度母——这种模糊感本身就是谜题的视觉基础。仁青提供的唐卡局部照片应有拍摄模糊感，部分区域有意过曝或阴影遮挡。

### 8.4 关键插画清单（优先级从高到低）

| 优先级 | 内容 |
|--------|------|
| P1（最高） | 旦增独坐院中，手持佛珠，嘴唇微动念六字真言，晨光打在他变形的手上 |
| P2（最高） | 经堂内部，空的画框，褪色壁画，翻倒的酥油灯 |
| P3（高） | 村口玛尼堆特写，石头刻着字，旁边散落风化牛骨 |
| P4（高） | 白玛独自蹲在破旧经堂外念经，衣着和普通老农无区别 |
| P5（中） | 五色经幡在山顶飘动：蓝白红绿黄从上到下 |
| P6（中） | 格桑在村口画素描的背影 |
| P7（中） | 旦增家院门口的藏獒，不同情绪状态需要五张 |
| P8（低） | 煨桑台，烟笔直升起时的仪式高潮场景 |

### 8.5 音效设计

| 场景/时刻 | 音效内容 |
|-----------|---------|
| 村子日常环境 | 奶牛叫声（非牦牛）、远处藏獒吠声、风声 |
| 经堂附近 | 经幡在风中的布料摩擦声、偶尔的转经筒金属转动声 |
| 旦增家院子 | 极安静，只有风声和极远处的念经声 |
| 仪式场景 | 铜钵声（低沉）、人声诵经声 |
| 夜晚 | 风声放大，远处偶有藏獒长吠 |
| 旦增说出真相时 | 所有声音消失，只剩风声 |

**特殊提示音：** 重要信息触发时，一声极短的低沉铜钵声。**全程不超过5次**，需与A、C协调使用在哪几个最关键节点。

**仪式成功时：** 铜钵声拉长，与风声混合，然后所有音效都安静下来，只剩风。

### 8.6 音乐方向

整体**无BGM，只有环境音**。

两个特殊时刻有音乐：
- **煨桑仪式成功：** 极简的藏族弦乐，空灵，不超过30秒，然后归于风声
- **结局滚动字幕：** 一段带人声低吟的藏族音乐，无具体词义，只有情绪

- 负责UI界面设计：对话框、道具栏、制作界面
- 负责音效素材收集与整合（环境音、铜钵声、仪式音乐）
- 参考章节：美术与音效规范

### 9.3 必须避开的五个坑

| 坑 | 说明 |
|----|------|
| **做太大** | 先做一个完整的单章，做好了再扩展。宁可一章做精，不要五章都粗糙 |
| **跳过纸上原型** | 在代码里改设计的成本是在纸上改的100倍，先用文字验证 |
| **AI对话喧宾夺主** | 如果玩家在聊AI本身而不是沉浸在文化故事里，设计就失败了 |
| **美术拖开发** | 先用占位图完成功能，美术替换可以在最后进行 |
| **不找人试玩** | 你觉得有趣的不一定别人觉得有趣。每个阶段都要找真实用户测试 |
